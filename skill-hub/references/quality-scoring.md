# Skill Hub 质量评级标准

## 评级体系

每个 Skill 按 5 个维度评分，总分 100 分，映射到 S/A/B/C 四级。

## 评分维度

### 1. 描述清晰度 (25分)

| 分数 | 标准 |
|------|------|
| 20-25 | frontmatter description 精确描述功能 + 包含多种触发关键词（中英文）+ 明确的使用场景 |
| 15-19 | 描述准确但触发词不够丰富，或缺少中文触发词 |
| 10-14 | 描述模糊，用户难以判断 Skill 的具体功能 |
| 0-9 | 描述缺失或与实际功能不符 |

### 2. 指令质量 (30分)

| 分数 | 标准 |
|------|------|
| 25-30 | 工作流程结构化（Step 1/2/3）、每步有明确的输入输出、包含边界情况处理、有示例 |
| 18-24 | 流程清晰但缺少边界处理或示例 |
| 10-17 | 有基本指令但缺乏结构，Agent 可能执行不一致 |
| 0-9 | 指令混乱或过于简单，Agent 难以执行 |

### 3. 资源完整度 (20分)

| 分数 | 标准 |
|------|------|
| 16-20 | 有 references/ 目录提供深度知识、有 scripts/ 提供自动化能力、资源与主文件紧密关联 |
| 11-15 | 有辅助资源但不够完整 |
| 6-10 | 仅有 SKILL.md，无辅助资源但 Skill 功能简单不需要 |
| 0-5 | 复杂功能但缺少必要的辅助资源 |

### 4. 触发精度 (15分)

| 分数 | 标准 |
|------|------|
| 12-15 | 触发条件明确，不会误触发其他场景，有 user-invocable 配置 |
| 8-11 | 触发条件基本准确，偶尔可能误触发 |
| 4-7 | 触发条件模糊，经常误触发或漏触发 |
| 0-3 | 无明确触发条件 |

### 5. 维护状态 (10分)

| 分数 | 标准 |
|------|------|
| 8-10 | 有版本号、最近 30 天内更新、有 changelog 或 commit history |
| 5-7 | 有版本号、最近 90 天内更新 |
| 3-4 | 超过 90 天未更新但功能稳定 |
| 0-2 | 无版本号、长期未维护、可能存在兼容性问题 |

## 评级映射

| 等级 | 总分 | 含义 | 推荐策略 |
|------|------|------|----------|
| **S** | 90-100 | 顶级质量 | 强烈推荐，优先展示 |
| **A** | 75-89 | 优秀 | 推荐使用 |
| **B** | 60-74 | 合格 | 可以使用，提示改进空间 |
| **C** | < 60 | 基础 | 不主动推荐，用户明确要求时展示并标注风险 |

## 快速评估流程

当无法获取完整信息时，可以使用简化评估：

1. 读取 SKILL.md 的 frontmatter → 评估维度 1 和 4
2. 读取 SKILL.md 的 body → 评估维度 2
3. 检查是否有 references/ 或 scripts/ → 评估维度 3
4. 检查 GitHub repo 的最后更新时间 → 评估维度 5

## 评级输出格式

```
质量评级: ⭐ A级 (82/100)
├── 描述清晰度: 22/25
├── 指令质量: 24/30
├── 资源完整度: 16/20
├── 触发精度: 12/15
└── 维护状态: 8/10
```
