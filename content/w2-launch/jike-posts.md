## Post 1: 痛点切入

Claude Code 的 Skill 生态有个大问题：80% 质量堪忧，还有一部分存在安全风险（curl | bash 管道执行、要求发送 Token 到不明服务器这种）。

我们做了一个东西叫 Skill Hub，本质是一个装在 Agent 里的应用市场。核心卖点两个：

1. 安装前自动执行 7 条安全红线扫描，有一条触发就拒绝
2. 每个 Skill 做 S/A/B/C 四级质量评分，低于 B 级不推荐

一行命令装 6 个精选 Skill：结构化头脑风暴（6 种框架）、设计系统生成器（出 Tailwind config + CSS variables）、小红书写手、视频脚本/分镜、飞书集成。

```
npx skills add second-me-01/skill-hub --all -g
```

装完之后对 Agent 说"帮我找一个做 XXX 的 Skill"就能用了。不自动安装任何东西，不收集数据。

## Post 2: 场景展示

最近在用 Claude Code 写小红书笔记，效果出乎意料。

装了一个叫 xhs-writer 的 Skill，它不是简单地"帮我写一篇"然后丢一坨 AI 味文字。它有完整的工作流：选题策划 → 5 个爆款标题（标注公式类型）→ 正文撰写（自动控制字数和节奏）→ 标签「3+3+3」策略 → 封面建议。

最实用的功能：自动违禁词检查。"最好""全网最低""根治"这些容易被限流的词，写的时候就帮你避开了。输出末尾还有完整的质检报告。

同一套工具包里还有视频脚本（输出分镜表格 + SRT 字幕）、结构化头脑风暴（SCAMPER、六顶帽、第一性原理等 6 种框架）、设计系统生成器、飞书 API 集成。

一行命令全装：
```
npx skills add second-me-01/skill-hub --all -g
```

## Post 3: 技术视角

做了一个给 Claude Code Skill 做安全审计的东西。

背景：Skill 本质是一段 Markdown 指令，安装到你的 Agent 后会影响 Agent 的行为。大部分人装 Skill 的时候不会去读源码。但我们扫了一圈发现不少 Skill 有问题：管道执行远程代码、要求关闭 SSL 验证、混淆的 Base64 代码块、甚至有 DAN 类越狱 prompt。

Skill Hub 的做法：在推荐任何第三方 Skill 前，自动执行 7 条安全规则的模式匹配扫描。同时有 5 维度质量评分体系（描述清晰度 25% + 指令质量 30% + 资源完整度 20% + 触发精度 15% + 维护状态 10%），映射到 S/A/B/C 四级。

目前自带 5 个高质量 Skill（4 个 S 级 + 1 个 A 级），涵盖头脑风暴、UI 设计系统、小红书、视频脚本、飞书。

```
npx skills add second-me-01/skill-hub --all -g
```

开源项目，代码全部可审查。
