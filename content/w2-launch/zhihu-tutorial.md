# Claude Code 必装的 6 个 Skill：让你的 AI Agent 能力翻倍

> 装 Skill 之前，先装 Skill Hub。

## 引言：你可能只用了 Claude Code 10% 的能力

如果你已经在用 Claude Code，恭喜你，你比 90% 的人都走在前面。

但我观察了身边大量的 Claude Code 用户后发现一个问题：**绝大多数人还在把它当一个高级补全工具**——写写代码、改改 bug、问问技术问题。这就像你买了一台顶配 MacBook Pro，但只用来上网和写文档。

Claude Code 的 Skill 系统才是它真正恐怖的地方。通过安装不同的 Skill，你可以让你的 Agent 获得全新的专业能力——写小红书爆款笔记、生成完整设计系统、做结构化头脑风暴、自动化飞书操作……这些能力是内置功能做不到的。

问题来了：**Skill 生态里 80% 是垃圾。**

我不是在夸张。随便搜一下你就会发现：大量 Skill 没有结构化的工作流程，描述含糊不清，甚至有些包含安全隐患——要求你 `curl | bash` 管道执行不明代码、要你把 API Key 发到来路不明的服务器。11% 左右的 Skill 存在可检测的安全风险。

这就是我们做 Skill Hub 的原因。

---

## Skill Hub 是什么

一句话：**Agent 时代的去中心化 Skill 应用市场，装在你的 Agent 里。**

它不是一个网站，不是一个浏览器插件。它本身就是一个 Skill——安装到你的 Claude Code 之后，它就成为你 Agent 内部的一个"应用商店"。你可以让 Agent 帮你搜索、验证、安装其他 Skill，全程在终端里完成。

核心机制有两个：

**1. 7 条安全红线扫描**

每个 Skill 在被推荐之前，必须通过 7 条安全检查。任何一条触发就直接拒绝：

- 禁止 `curl | bash` 等管道执行命令
- 禁止要求提供/发送 API Key、Token、密码等凭证到外部
- 禁止关闭安全功能（如 `--no-verify`、`--insecure`）
- 禁止 Base64 编码或混淆代码
- 禁止修改系统文件（`/etc/`、`~/.ssh/` 等）
- 禁止未告知的数据收集
- 禁止 AI 安全绕过指令（越狱 prompt 等）

这不是口号。每一条规则都有具体的检测模式和正则匹配逻辑。

**2. S/A/B/C 四级质量评分**

通过安全检查后，Skill Hub 会从 5 个维度对 Skill 打分：

| 维度 | 权重 | 评估内容 |
|------|------|---------|
| 描述清晰度 | 25% | frontmatter 描述是否准确、触发关键词是否丰富 |
| 指令质量 | 30% | 工作流程是否结构化、可执行、有边界处理 |
| 资源完整度 | 20% | 是否有 references/scripts 等辅助资源 |
| 触发精度 | 15% | 是否有明确触发条件，避免误触发 |
| 维护状态 | 10% | 版本号、更新频率 |

S 级（90-100 分）强烈推荐，C 级（< 60 分）不主动推荐。低于 B 级的 Skill，Skill Hub 根本不会给你看。

## 安装：一行命令

```bash
npx skills add secondme-team/skill-hub --all -g
```

这一行命令会安装全部 6 个 Skill。你也可以单独安装某一个：

```bash
# 只装核心 Skill Hub
npx skills add secondme-team/skill-hub@skill-hub -g

# 只装某个卫星 Skill
npx skills add secondme-team/skill-hub@smart-brainstorm -g
```

安装完成后，这些 Skill 会常驻在你的 Claude Code 里，根据对话内容自动触发，或者你可以通过关键词主动调用。

## 6 个 Skill 逐一介绍

### 1. skill-hub — Meta Skill：发现 / 安全检查 / 安装

这是整个体系的核心。安装它之后，你的 Agent 就拥有了"找 Skill 并安装"的能力。

**使用场景**：当你需要一个 Claude Code 目前没有的能力时，直接说：
- "帮我找一个做 XXX 的 Skill"
- "有没有 Skill 可以 XXX"
- "install skill for XXX"

Skill Hub 会自动搜索精选推荐库和公开生态，对候选 Skill 执行安全扫描和质量评级，然后以结构化的方式展示给你。你确认后一键安装。

它绝不会自动安装任何东西，必须你确认。也不会收集你的搜索记录。

### 2. smart-brainstorm — 结构化头脑风暴引擎

这不是那种"帮我想 10 个点子"然后给你一堆平庸清单的工具。

Smart Brainstorm 内置 6 种经过验证的思维框架：

| 框架 | 最适合场景 |
|------|-----------|
| SCAMPER | 改进现有产品/流程 |
| 六顶思考帽 | 团队决策，需要多角度评估 |
| 第一性原理 | 颠覆性创新，打破行业惯例 |
| SWOT 分析 | 战略规划，竞争分析 |
| How Might We | 用户体验，服务设计 |
| 逆向头脑风暴 | 陷入僵局，常规方法失效 |

**完整工作流程**：问题定义 → 框架选择 → 发散执行 → 创意筛选（5 维评估：影响力/可行性/新颖性/速度/风险）→ 行动计划（含里程碑和风险预案）。

有一条质量红线我特别喜欢：**禁止空话套话**。"提升用户体验""加强品牌建设"这种不含具体行动的描述，不允许出现在输出里。每个创意必须可执行，至少能回答"第一步做什么"。

**使用场景**：当你说"帮我想想"、"brainstorm"、"头脑风暴"时自动触发。

### 3. ui-design-system — 专业级设计系统生成器

给它一个品牌色，它能生成一整套生产级设计系统：

- **Design Tokens**：完整的色彩体系（11 级色阶，含语义色）、排版阶梯、间距系统（4px 网格）、阴影、圆角、断点
- **组件模式**：Button、Input、Card、Modal、Navigation、Table、Form、Toast 等，每个组件含结构/状态/变体/无障碍规范
- **三种输出格式**：Tailwind CSS Config、CSS Custom Properties、Vanilla CSS
- **深色模式**：每个 token 自带 light/dark 双模式
- **无障碍合规**：WCAG 2.1 AA 标准，含对比度检查、键盘导航、ARIA 标注
- **中英文排版**：内置 CJK + Latin 双文字栈

你说"帮我生成一个设计系统"，它就会直接生成，不会反复追问你。所有 token 值旁边都有决策说明（比如"对比度 4.7:1，满足 WCAG AA"）。

**使用场景**：开新项目需要设计规范、给已有项目补设计系统、前端需要一套标准的组件模式。

### 4. xhs-writer — 小红书爆款内容创作助手

覆盖小红书创作全流程：**选题策划 → 标题创作 → 正文撰写 → 标签策略 → 封面建议**。

几个亮点：

- **标题创作**：生成 5 个备选标题，每个标注使用的爆款公式（数字清单型/反常识型/痛点解决型/身份共鸣型/对比冲突型），并推荐最佳方案
- **标签策略**：「3+3+3」法则——3 个热门大标签 + 3 个精准中标签 + 3 个长尾小标签
- **违禁词自动检查**：主动规避绝对化用语、虚假宣传词、引流违规词、医疗/金融敏感词
- **自动质检**：输出末尾附带完整检查报告（标题字数、正文字数、emoji 密度、违禁词扫描、标签数量、互动引导、预估表现）

输出的笔记可以直接复制到小红书发布，零修改成本。

**使用场景**：说"帮我写小红书笔记"、"小红书标题怎么起"即可触发。

### 5. video-script — 全流程视频脚本创作助手

支持抖音、B站、YouTube、TikTok 四个平台，支持口播、情景剧、教程、Vlog、带货五种视频类型。

核心输出是**结构化分镜表格**：

| 镜号 | 时长 | 景别/运镜 | 画面描述 | 台词/旁白 | 字幕/花字 | 备注 |
|------|------|----------|---------|----------|----------|------|

还能生成 SRT 格式字幕文件（可直接导入剪辑软件）、拍摄提示（机位设置、灯光建议、收音建议、道具清单）、发布建议。

口播稿会标注重音（加粗）和停顿标记 `[停顿1s]`，按中文口播语速每分钟 250-280 字控制时长。

进阶功能包括**系列化策划**（5-10 期选题大纲）和 **A/B 标题测试**（悬念型/数据型/痛点型）。

**使用场景**：说"帮我写视频脚本"、"分镜设计"、"视频文案"时触发。

### 6. feishu-kit — 飞书 API 集成工具包

覆盖飞书开放平台 5 大常用场景：

1. **消息发送**：文本/富文本/卡片消息，支持群组和个人
2. **文档操作**：创建文档、读取内容、创建/读取电子表格
3. **多维表格**（Bitable）：增删改查记录，批量操作
4. **Webhook 机器人**：最简消息推送，无需 token
5. **审批查询**：查询审批实例列表和详情

附带 shell 辅助脚本，加载后一行命令完成操作：

```bash
source scripts/feishu-api.sh
send_webhook_message "部署完成，版本 v1.2.3"
```

内置完整错误码对照表和排障流程。所有凭证通过环境变量传递，不硬编码。

**使用场景**：说"飞书发消息"、"操作飞书文档"、"飞书机器人"时触发。

## 为什么选 Skill Hub 而不是自己找

三个原因：

1. **安全**——你不需要自己审查每个 Skill 的代码是否有坑。7 条安全红线帮你过滤。
2. **质量**——S/A/B/C 评级帮你在海量 Skill 中快速定位高质量的。低于 B 级的你根本看不到。
3. **效率**——不用离开终端。对 Agent 说一句话就能搜索、验证、安装，全流程 30 秒搞定。
